# Huayijun：多邻省 AI 学习助手

### 本节课最适合先做的版本

本节课不建议直接做完整学习平台、语音播放和学习进度系统，更适合先完成“**方言转换与练习生成器**”。

建议本节课的 MVP 收缩为：

- 输入：普通话句子、目标方言、学习模式
- 输出：方言表达、解释、练习题

### 建议的最小功能

1. 页面支持输入：
   - 原句
   - 目标方言
   - 模式，例如“学习/练习/测试”
2. 后端调用模型
3. 输出采用固定结构，例如：
   - `dialect_output`
   - `pronunciation_note`
   - `usage_explanation`
   - `practice_question`
   - `common_mistake`

### Prompt 工程建议

- 提示词中明确告诉模型：如果没有把握，不要伪造“绝对标准答案”
- 要求区分“更常见表达”“可能地区差异”
- 先聚焦 1 到 2 个方言区，不要一次覆盖全部方言
- 用少量 few-shot 示例固定输出风格

### 本节课不建议做的内容

- 语音合成
- 语音识别
- 多方言大知识库
- 个性化学习路径系统

### 课堂验证建议

建议准备：

1. 一个日常问候样本
2. 一个购物/问路场景样本
3. 一个容易引起地区差异争议的样本

重点验证：

- 输出是否自然
- 是否清楚标注了不确定性
- 练习题是否与输入内容对应
