# Project One-Pager(v1)

## 1. 项目基本信息

- **项目名称**：多邻省 AI 学习助手
- **制作人姓名**：*你猜（bushi
- **目标用户（谁会使用）**：那些不会方言（但又需要用）的
- **场景一句话（在哪个场景解决什么问题）**：在学习方言有困难时，可以使用此助手学习和练习。

## 2. 问题定义

### 当前痛点：

1. 方言学习者缺乏个性化的学习材料和即时反馈
2. 传统方言学习资源枯燥，缺少互动性和趣味性
3. 不同地区方言差异大，难以找到针对性的学习内容

### 项目边界（本项目不做什么）：

- 不提供专业的语言学家级别的方言准确性保证
- 不支持所有方言（初期聚焦主要方言区）
- 不替代真人语言教师，而是作为辅助学习工具

### 成功标准（如何判断项目有效）：

- 用户能够完成基础方言对话练习并获得准确反馈
- 用户对学习内容的满意度评分达到 4/5 以上
- 系统响应时间在可接受范围内（< 3 秒）

## 3. 最小可行方案（MVP）

### 输入是什么：

- 用户输入的文本（普通话或方言）
- 用户选择的目标方言类型
- 学习模式选择（学习/练习/测试）

### 输出是什么：

- 方言翻译或转换结果
- 发音指导和解释
- 练习题和反馈
- 学习进度建议

### 核心流程（3-6 步）：

1. 用户输入要学习的语句或选择学习场景
2. 系统识别并转换为目标方言
3. 提供方言的发音规则和文化背景说明
4. 生成练习题供用户巩固
5. 根据用户表现提供个性化反馈和改进建议

## 4. 技术方案草图

### 模型与环境：

使用`Ollama + qwen3.5:4b`

### Prompt 方案：

- 设计方言转换专用 prompt，包含方言特征描述和示例
- 使用 few-shot learning 提供高质量方言对照样本
- 设计交互式学习 prompt，引导用户逐步掌握方言

### RAG 方案（若暂不做请写"后续阶段"）：

- 构建方言知识库，包含词汇、语法、发音规则
- 检索相关方言知识增强模型回答准确性
- 支持动态更新方言数据
- Agent / Tool 方案（若暂不做请写"后续阶段"）：
  - 实现学习进度追踪工具
  - 集成语音播放工具（可选）
  - 提供练习题生成和批改工具
- 安全与日志方案（最小版）：
  - 记录用户交互日志用于改进
  - 过滤不当内容和敏感话题
  - 保护用户隐私数据

## 5. 数据与资料来源

### 数据来源列表：

1. 公开方言语料库和语言学研究报告
2. 网络方言学习资源和社区贡献内容
3. 地方志和方言词典数字化资料

### 数据质量风险：

- 方言数据可能存在地区偏差，不够全面
- 部分方言资料时效性较差，需要验证
- 网络资源质量参差不齐，需要筛选

## 6. 评测计划（v1）

### 关键指标：

1. 方言转换准确率（与标准方言对比）
2. 用户学习效果和满意度评分

### 评测样本如何构建：

- 收集常见日常对话场景（问候、购物、问路等）
- 覆盖不同难度等级的语句
- 邀请目标用户群体进行测试

### 失败样本如何记录：

- 记录转换错误的案例及原因分析
- 收集用户反馈的不满意回答
- 建立错误案例库用于迭代优化

## 7. 里程碑计划

### M1：Prompt + 基线评测要达到什么结果：

- 完成基础方言转换 prompt 设计
- 实现简单的问答交互功能
- 基线评测准确率达到 60% 以上

### M2：RAG 要达到什么结果：

- 构建方言知识库原型
- 实现 RAG 检索增强功能
- 准确率提升至 75% 以上

### M3：Agent + Tool + Safety 要达到什么结果：

- 实现学习进度追踪和个性化推荐
- 集成安全防护机制
- 完整的学习闭环功能

### M4：Capstone 展示包要包含什么：

- 完整的演示应用
- 技术文档和用户手册
- 评测报告和案例分析
- 未来改进路线图

## 8. 风险与备选方案

- 主要风险（最多 3 个）：

1. 方言数据质量不足，影响转换准确性
2. 模型对方言理解能力有限
3. 开发进度可能滞后

- 对应备选方案：

1. 多渠道收集数据，人工校验关键内容；优先保证核心场景质量
2. 结合规则引擎和 RAG 提升准确性；明确能力边界，管理用户预期
3. 采用敏捷开发，优先实现 MVP；合理分配时间，重点突破核心功能
